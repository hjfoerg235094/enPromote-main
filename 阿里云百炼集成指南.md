# 阿里云百炼集成指南

## 概述

本文档详细说明如何在enPromote项目中集成阿里云百炼AI服务，实现智能题目生成功能。

## 前置要求

1. 阿里云账号
2. 开通阿里云百炼服务
3. 获取API Key

## 配置步骤

### 1. 获取阿里云百炼API Key

1. 登录阿里云控制台
2. 进入"百炼"服务页面
3. 在"API-KEY管理"中创建新的API Key
4. 保存API Key（仅显示一次，请妥善保管）

### 2. 配置API Key

编辑 `server/config/aliyun.js` 文件：

```javascript
const aliyunConfig = {
  // 将 'your-api-key-here' 替换为你的实际API Key
  apiKey: 'sk-your-actual-api-key-here',

  // API端点
  endpoint: 'https://dashscope.aliyuncs.com/compatible-mode/v1',

  // 模型配置
  models: {
    turbo: 'qwen-turbo',    // 快速、经济
    plus: 'qwen-plus',      // 平衡性能和成本（推荐）
    max: 'qwen-max',        // 最强性能
    long: 'qwen-long'       // 长文本
  },

  // 默认模型
  defaultModel: 'qwen-plus'
};
```

### 3. 验证配置

在 `server/router/question.js` 中，阿里云百炼客户端已经配置好：

```javascript
const aliyunClient = new OpenAI({
  baseURL: aliyunConfig.endpoint,
  apiKey: aliyunConfig.apiKey,
  defaultHeaders: {
    'X-DashScope-SSE': 'disable'  // 禁用SSE流式响应
  }
});
```

## 使用方法

### 前端调用

在前端组件中调用题目生成API时，指定使用阿里云百炼：

```typescript
import { generateQuestions } from '@/api/question';

// 使用阿里云百炼生成题目
const response = await generateQuestions({
  chapter: 'A',           // 章节：A-酒店场景，B-餐厅场景
  level: 4,              // 关卡：第四关（情景对话）
  aiProvider: 'aliyun'    // 指定使用阿里云百炼
});
```

### 后端处理流程

后端接收到请求后：

1. 验证AI提供商参数（确保是 'aliyun'）
2. 获取用户已学词汇（优先级最高的10个单词）
3. 构建AI提示词
4. 调用阿里云百炼API生成题目
5. 解析AI返回的JSON
6. 保存题目到MongoDB数据库

## 模型选择

阿里云百炼提供多种模型，可根据需求选择：

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| qwen-turbo | 快速、经济 | 大量题目生成、快速测试 |
| qwen-plus | 平衡性能和成本 | 生产环境推荐使用 |
| qwen-max | 最强性能 | 高质量题目生成 |
| qwen-long | 长文本处理 | 复杂场景对话生成 |

在 `server/config/aliyun.js` 中修改 `defaultModel` 可切换默认模型。

## 题目质量评估

### 评估维度

1. **题目准确性**：题目内容是否正确
2. **难度适配性**：是否符合关卡难度要求
3. **词汇相关性**：是否合理使用已学词汇
4. **场景真实性**：是否符合实际应用场景
5. **答案解析质量**：解析是否清晰准确

### 评估方法

1. 自动评估：
   - 检查JSON格式是否正确
   - 验证必填字段是否存在
   - 检查选项数量是否正确

2. 人工评估：
   - 定期抽查生成题目
   - 收集用户反馈
   - 统计答题正确率

## 常见问题

### 1. API调用失败

**问题**：返回 "API Key无效" 错误

**解决方案**：
- 检查API Key是否正确配置
- 确认API Key是否已激活
- 检查账户余额是否充足

### 2. 生成题目格式错误

**问题**：返回的JSON无法解析

**解决方案**：
- 检查提示词是否完整
- 尝试使用更稳定的模型（如 qwen-plus）
- 增加JSON格式约束的提示

### 3. 生成速度慢

**问题**：题目生成耗时过长

**解决方案**：
- 使用 qwen-turbo 模型提高速度
- 减少单次生成的题目数量
- 实现异步生成机制

### 4. 题目质量不稳定

**问题**：生成题目质量参差不齐

**解决方案**：
- 优化提示词模板
- 使用更高质量的模型（qwen-max）
- 添加题目后处理逻辑

## 最佳实践

1. **模型选择**
   - 开发测试阶段：使用 qwen-turbo 快速迭代
   - 生产环境：使用 qwen-plus 平衡性能和成本
   - 高质量需求：使用 qwen-max

2. **错误处理**
   - 添加重试机制
   - 实现降级方案（切换到备用模型）
   - 记录详细日志便于排查问题

3. **性能优化**
   - 实现题目缓存
   - 批量生成题目
   - 使用异步处理

4. **成本控制**
   - 监控API调用次数和成本
   - 合理设置题目生成频率
   - 优化提示词长度

## 扩展功能

### 1. 支持更多关卡

在 `server/router/question.js` 中添加更多关卡提示词模板：

```javascript
const LEVEL_PROMPTS = {
  1: (chapter, scenario, wordList) => `第一关提示词...`,
  2: (chapter, scenario, wordList) => `第二关提示词...`,
  3: (chapter, scenario, wordList) => `第三关提示词...`,
  4: LEVEL_4_PROMPT_TEMPLATE,
  5: (chapter, scenario, wordList) => `第五关提示词...`
};
```

### 2. 添加题目类型

系统已支持以下题目类型：

**基础题型**：
- 选择题（multiple_choice）
- 填空题（fill_in_blank）
- 情景对话题（scenario）

**扩展题型**：
- 听力理解题（listening_comprehension）
  - 包含听力材料文本
  - 根据内容生成2-3道理解题
  - 每题包含4个选项
  
- 完形填空题（cloze_test）
  - 创建短文并挖空
  - 每个空提供4个备选词
  - 提供完整答案解析
  
- 翻译题（translation）
  - 提供英语句子和中文翻译
  - 包含翻译技巧说明
  
- 写作题（writing）
  - 设定写作任务和要求
  - 提供评分标准
  - 包含范文示例

系统会从以上7种题型中随机选择3-5种生成，确保每次生成的题目组合不同，增加学习趣味性。

### 3. 实现题目质量评估

添加题目质量评估机制：
- 用户反馈
- 答题正确率统计
- AI生成质量评分

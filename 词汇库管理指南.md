## 系统架构

词汇库系统采用JSON文件+MongoDB混合架构：

- **JSON文件**：存储词汇基础数据，按章节（A-Z）组织
- **MongoDB**：存储用户学习进度和单词详细信息

### 主要组件

1. **vocabularyService**：词汇服务模块，提供词汇数据的获取和处理
2. **vocabularyDataManager**：词汇数据管理工具，用于数据标准化
3. **vocabularyDbOptimizer**：数据库优化工具，用于优化数据库结构和性能
4. **vocabulary路由**：提供词汇相关的API接口

## 数据标准化

### 词汇数据格式

标准化的词汇数据格式如下：

```json
{
  "word": "单词",
  "mean": "中文释义",
  "phonetic_symbol": "音标",
  "initial": "首字母",
  "sence": "章节",
  "difficulty": 1,
  "tags": [],
  "examples": []
}
```

### 数据标准化步骤

1. **运行数据管理工具**：

```bash
cd server
node scripts/vocabularyDataManager.js
```

2. **查看质量报告**：

工具会为每个词汇文件生成质量报告，包括：
- 总单词数
- 有效单词数
- 无效单词数
- 数据完整率
- 各章节详情
- 问题列表

3. **处理问题数据**：

根据质量报告中的问题列表，手动修复或删除无效数据。

### 数据标准化功能

词汇数据管理工具提供以下功能：

- **normalizeWordData**：标准化单词数据格式
- **validateWordData**：验证单词数据有效性
- **processVocabularyFile**：处理词汇文件
- **mergeVocabularyFiles**：合并多个词汇文件
- **analyzeVocabularyQuality**：分析词汇数据质量
- **generateQualityReport**：生成质量报告

## 数据导入

### 导入JSON词汇数据到MongoDB

1. **运行数据导入工具**：

```bash
cd server
node scripts/vocabularyDataImporter.js
```

2. **导入结果**：

工具会显示导入统计信息，包括：
- 总计处理单词数
- 新导入单词数
- 更新单词数
- 跳过单词数
- 错误列表

### 数据导入功能

词汇数据导入工具提供以下功能：

- **loadVocabularyFromFile**：从JSON文件加载词汇数据
- **transformWordData**：转换单词数据格式
- **importWord**：导入单个单词
- **importVocabularyFile**：导入词汇文件
- **importAllVocabularyFiles**：导入所有词汇文件

### 数据导入示例

```javascript
const VocabularyDataImporter = require('./scripts/vocabularyDataImporter');
const importer = new VocabularyDataImporter();

// 连接数据库
await importer.connect();

// 导入所有词汇文件
await importer.importAllVocabularyFiles();

// 导入特定词汇文件
await importer.importVocabularyFile('./vocabulary/CET-4.json', 'CET-4');

// 关闭连接
await importer.close();
```

## 数据导出

### 从MongoDB导出词汇数据到JSON

1. **运行数据导出工具**：

```bash
cd server
node scripts/vocabularyDataExporter.js
```

2. **导出结果**：

工具会显示导出统计信息，包括：

### 数据导出功能

词汇数据导出工具提供以下功能：

- **transformWordData**：转换单词数据格式
- **exportVocabulary**：导出词汇数据到JSON文件
- **exportAllVocabulary**：导出所有词汇数据
- **exportChapter**：导出指定章节的词汇数据

### 数据导出示例

```javascript
const VocabularyDataExporter = require('./scripts/vocabularyDataExporter');
const exporter = new VocabularyDataExporter();

// 连接数据库
await exporter.connect();

// 导出所有词汇数据
await exporter.exportAllVocabulary();

// 导出特定等级的词汇数据
await exporter.exportVocabulary('CET-4', './vocabulary/CET-4_exported.json');

// 导出特定章节的词汇数据
await exporter.exportChapter('CET-4', 'A', './vocabulary/CET-4_A.json');

// 关闭连接
await exporter.close();
```

## 词汇库管理工具

### 工具简介

词汇库管理工具（vocabularyManager.js）是一个命令行工具，整合了词汇数据管理的所有功能，提供统一的接口来执行常见的词汇库管理任务。

### 安装依赖

在使用词汇库管理工具之前，需要安装commander包：

```bash
cd server
npm install commander --save
```

### 基本用法

```bash
cd server
node scripts/vocabularyManager.js [command] [options]
```

### 可用命令

#### 1. 数据标准化命令

```bash
node scripts/vocabularyManager.js normalize [options]
```

**选项**：
- `-f, --file <file>` - 指定要处理的词汇文件
- `-a, --all` - 处理所有词汇文件

**示例**：
```bash
# 处理所有词汇文件
node scripts/vocabularyManager.js normalize --all

# 处理特定词汇文件
node scripts/vocabularyManager.js normalize --file ./vocabulary/CET-4.json
```

#### 2. 数据质量分析命令

```bash
node scripts/vocabularyManager.js analyze [options]
```

**选项**：
- `-f, --file <file>` - 指定要分析的词汇文件

**示例**：
```bash
# 分析特定词汇文件
node scripts/vocabularyManager.js analyze --file ./vocabulary/CET-4.json
```

#### 3. 数据库优化命令

```bash
node scripts/vocabularyManager.js optimize [options]
```

**选项**：
- `-i, --index` - 只创建索引
- `-d, --data` - 只优化数据

**示例**：
```bash
# 全面优化
node scripts/vocabularyManager.js optimize

# 只创建索引
node scripts/vocabularyManager.js optimize --index

# 只优化数据
node scripts/vocabularyManager.js optimize --data
```

#### 4. 数据导入命令

```bash
node scripts/vocabularyManager.js import [options]
```

**选项**：
- `-f, --file <file>` - 指定要导入的词汇文件
- `-l, --level <level>` - 指定词汇等级
- `-a, --all` - 导入所有词汇文件

**示例**：
```bash
# 导入所有词汇文件
node scripts/vocabularyManager.js import --all

# 导入特定词汇文件
node scripts/vocabularyManager.js import --file ./vocabulary/CET-4.json --level CET-4
```

#### 5. 数据导出命令

```bash
node scripts/vocabularyManager.js export [options]
```

**选项**：
- `-l, --level <level>` - 指定词汇等级
- `-c, --chapter <chapter>` - 指定章节
- `-o, --output <file>` - 指定输出文件路径
- `-a, --all` - 导出所有词汇数据

**示例**：
```bash
# 导出所有词汇数据
node scripts/vocabularyManager.js export --all

# 导出特定等级的词汇数据
node scripts/vocabularyManager.js export --level CET-4

# 导出特定章节的词汇数据
node scripts/vocabularyManager.js export --level CET-4 --chapter A --output ./vocabulary/CET-4_A.json
```

#### 6. 帮助命令

```bash
node scripts/vocabularyManager.js help
```

### 常用操作流程

#### 初始化词汇库

```bash
# 1. 标准化词汇数据
node scripts/vocabularyManager.js normalize --all

# 2. 分析数据质量
node scripts/vocabularyManager.js analyze --file ./vocabulary/CET-4.json

# 3. 导入数据到数据库
node scripts/vocabularyManager.js import --all

# 4. 优化数据库
node scripts/vocabularyManager.js optimize
```

#### 定期维护

```bash
# 1. 优化数据库
node scripts/vocabularyManager.js optimize

```

#### 更新词汇数据

```bash

node scripts/vocabularyManager.js export --all

# 2. 手动编辑词汇文件

# 3. 标准化词汇数据
node scripts/vocabularyManager.js normalize --all

node scripts/vocabularyManager.js import --all

node scripts/vocabularyManager.js optimize
```

### 索引优化

工具会创建以下索引：

**Word模型索引**：
- 单词文本索引（唯一）
- 章节索引
- 场景索引
- 等级索引
- 复合索引（章节+场景）

**UserWord模型索引**：
- 用户ID和单词ID复合索引（唯一）
- 用户ID索引
- 单词ID索引
- 状态索引
- 优先级索引
- 下次复习时间索引
- 复合索引（用户ID+下次复习时间）

### 数据清理

工具会执行以下数据清理操作：

- 删除重复的单词记录
- 删除无效的UserWord记录
- 重置过期的复习记录

## API使用

### 基础URL

```
http://your-domain/api/vocabulary
```

### API端点

#### 1. 获取单词列表

```
GET /api/vocabulary/words
```

**查询参数**：
- `level`: 词汇等级 (CET-4, CET-6, PEE等)
- `chapter`: 章节字母 (A-Z)
- `start`: 起始索引 (默认0)
- `limit`: 返回数量限制 (默认20)

**示例**：
```
GET /api/vocabulary/words?level=CET-4&chapter=A&start=0&limit=20
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "words": [...],
    "total": 100,
    "hasMore": true,
    "currentIndex": 0
  }
}
```

#### 2. 获取单词详细信息

```
GET /api/vocabulary/word/:word
```

**示例**：
```
GET /api/vocabulary/word/abandon
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "word": "abandon",
    "phonetics": [...],
    "chineseMeaning": "...",
    "meanings": [...]
  }
}
```

#### 3. 批量获取单词详细信息

```
POST /api/vocabulary/words/batch
```

**请求体**：
```json
{
  "words": ["word1", "word2", ...]
}
```

**响应**：
```json
{
  "code": 200,
  "data": [...]
}
```

#### 4. 搜索单词

```
GET /api/vocabulary/search
```

**查询参数**：
- `keyword`: 搜索关键词
- `level`: 词汇等级 (默认CET-4)

**示例**：
```
GET /api/vocabulary/search?keyword=ab&level=CET-4
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "keyword": "ab",
    "level": "CET-4",
    "results": [...],
    "count": 10
  }
}
```

#### 5. 获取用户学习进度

```
GET /api/vocabulary/progress
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "total": 100,
    "learned": 50,
    "learning": 30,
    "difficult": 20,
    "progress": 50
  }
}
```

#### 6. 更新用户单词学习状态

```
POST /api/vocabulary/word/status
```

**请求体**：
```json
{
  "wordId": "...",
  "status": "know",
  "isCorrect": true
}
```

**响应**：
```json
{
  "code": 200,
  "message": "更新成功",
  "data": {...}
}
```

#### 7. 获取需要复习的单词

```
GET /api/vocabulary/review
```

**响应**：
```json
{
  "code": 200,
  "data": {
    "words": [...],
    "count": 10
  }
}
```

#### 8. 清除词汇缓存

```
POST /api/vocabulary/cache/clear
```

**请求体**：
```json
{
  "level": "CET-4"  // 可选
}
```

**响应**：
```json
{
  "code": 200,
  "message": "缓存清除成功"
}
```

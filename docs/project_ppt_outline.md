# 英语学习系统 PPT 展示内容

## 1. 项目基础与研究背景

- 当前英语学习普遍存在“背了就忘”的问题，学习过程缺少持续复习和记忆状态追踪。
- 传统学习工具多以固定词表和统一任务为主，难以根据用户掌握情况动态调整学习内容。
- 英语学习中的听、说、读、写往往被割裂，用户难以把单词真正迁移到真实表达场景中。
- 本项目以 Vue3 + Node.js + MongoDB 构建完整学习系统，将词汇学习、复习、AI 对话和学习记录统一到一个闭环中。
- 系统通过单词掌握状态、复习次数、下次复习时间等数据，提升复习安排的科学性。
- AI 能生成对话、反馈和场景任务，使学习从“记忆单词”扩展到“真实表达训练”。
- 项目意义在于提高学习效率、增强个性化体验，并探索 AI 在英语学习产品中的实际落地方式。

## 2. 系统设计与关键技术

- 系统采用前后端分离架构，前端负责学习交互与状态展示，后端负责业务逻辑、数据存储和 AI 服务调用。
- 前端使用 Vue3、Vue Router、Pinia 和 Axios，构建词汇学习、复习卡片、AI 对话、剧情任务等页面。
- 后端基于 Node.js + Express，提供用户、单词、复习计划、AI 对话、学习记录等 REST API。
- 数据层使用 MongoDB，存储用户信息、单词库、用户单词状态、学习记录、剧情进度和对话记录。
- 单词学习采用状态标记机制，如 unknown、learning、know、mastered，用于判断用户掌握程度。
- 复习机制结合 reviewCounts、priority、nextReviewTime 等字段，实现基于表现的复习优先级调整。
- AI 模块通过大模型接口生成对话内容、学习反馈和场景表达，使学习内容更贴近用户当前任务。

## 3. 产品功能与竞品优势

- 系统核心功能包括词汇学习、闪卡复习、AI 口语陪练、AI 对话练习、剧情闯关、学习记录和好友互动。
- 词汇学习模块不仅展示释义和音标，还能结合例句生成和复习状态更新，形成持续学习链路。
- 复习模块根据用户答题表现动态调整单词状态，避免所有单词按固定频率重复出现。
- AI 对话模块支持用户在具体场景中练习表达，而不是只停留在单词记忆层面。
- AI 口语陪练通过录音评测、得分反馈和低分片段复练，增强口语训练的即时反馈能力。
- 相比百词斩、扇贝等传统背单词产品，本项目更强调“单词到表达”的迁移训练。
- AI 带来的差异化主要体现在个性化反馈、场景化任务生成和根据学习表现继续推进内容。

## 4. 研究总结

- 本项目的创新点在于将词汇记忆、复习算法、AI 对话和口语评测整合为一个学习闭环。
- 系统不只记录“学了什么”，还记录用户掌握状态、复习表现和学习时长，为后续推荐提供数据基础。
- AI 的引入让学习内容具备动态生成能力，减少固定题库和固定对话带来的重复感。
- 当前不足是 AI 生成内容质量依赖接口稳定性，部分功能仍需要更严格的结果校验和容错机制。
- 口语训练目前仍以目标句评测为主，后续可进一步接入 ASR，实现自由表达识别与评价。
- 后续可优化智能推荐算法，根据遗忘风险、错误类型和学习目标生成个性化学习路径。
- 未来还可以扩展更多真实场景，如面试、旅行、校园、商务沟通，提高系统的应用价值。

## 质量评估数据说明

### 三个维度

1. **记忆能力** (Memory)
   - 得分计算: (正确率 × 0.6) + (记忆保持率 × 0.4)
   - 指标:
     - 正确率: 拼写练习的平均正确率
     - 记忆保持率: 已掌握单词数 / 总学习单词数 × 100
     - 已掌握: 复习次数 >= 3 的单词

2. **理解能力** (Understanding)
   - 得分计算: 听力练习的平均完成度
   - 指标:
     - 理解度: 听力练习的平均完成度

3. **应用能力** (Application)
   - 得分计算: AI对话练习次数 × 10（上限100）
   - 指标:
     - 应用能力: 平均AI对话练习次数

### 综合得分

- 计算公式: (记忆能力 × 0.4) + (理解能力 × 0.3) + (应用能力 × 0.3)
- 权重说明:
  - 记忆能力占40%（最重要）
  - 理解能力占30%
  - 应用能力占30%

### 得分等级

- 90-100分: 优秀
- 80-89分: 良好
- 60-79分: 一般
- 0-59分: 需改进

## 数据来源

质量评估的数据来源：

1. **StudyRecord集合**
   - 学习记录
   - 包含拼写正确率、听力完成度、AI练习次数等

2. **UserWord集合**
   - 用户单词学习记录
   - 包含复习次数、首次学习时间等

## 计算逻辑详解

### 记忆能力计算

```javascript
// 1. 获取平均正确率
const avgAccuracy = records.reduce((sum, r) => 
  sum + r.practiceStats.spellingAccuracy, 0) / records.length;

// 2. 计算记忆保持率
const masteredWords = userWords.filter(w => w.reviewCounts >= 3).length;
const retentionRate = (masteredWords / userWords.length) * 100;

// 3. 计算记忆能力得分
const memoryScore = (avgAccuracy * 0.6) + (retentionRate * 0.4);
```

### 理解能力计算

```javascript
// 获取听力完成度平均值
const avgListeningCompletion = records.reduce((sum, r) => 
  sum + r.practiceStats.listeningCompletion, 0) / records.length;

// 理解能力得分 = 听力完成度
const understandingScore = avgListeningCompletion;
```

### 应用能力计算

```javascript
// 获取AI对话次数平均值
const avgAiPracticeCount = records.reduce((sum, r) => 
  sum + r.practiceStats.aiPracticeCount, 0) / records.length;

// 应用能力得分 = AI对话次数 × 10（上限100）
const applicationScore = Math.min(avgAiPracticeCount * 10, 100);
```

### 综合得分计算

```javascript
const overallScore = (memoryScore * 0.4) + 
                   (understandingScore * 0.3) + 
                   (applicationScore * 0.3);
```

## 质量改进建议生成

系统会根据各维度得分自动生成建议：

### 记忆能力建议

- 触发条件: 记忆能力得分 < 70
- 建议: 使用间隔重复法复习
- 优先级: 高

### 理解能力建议

- 触发条件: 理解能力得分 < 70
- 建议: 多阅读英文文章
- 优先级: 中

### 应用能力建议

- 触发条件: 应用能力得分 < 70
- 建议: 通过AI对话、造句等方式加强应用
- 优先级: 高

## 注意事项

1. **数据准确性**
   - 确保在学习模块中正确记录数据
   - 正确率应该基于实际答题情况
   - 复习次数应该准确记录

2. **数据完整性**
   - 需要有足够的学习记录才能计算准确
   - 建议至少有7天的学习数据

3. **性能优化**
   - 考虑添加数据缓存
   - 优化数据库查询
   - 定期清理旧数据

4. **用户体验**
   - 没有数据时显示友好的提示
   - 提供明确的改进建议
   - 建议应该可执行

## 常见问题

### Q1: 为什么质量得分都是0？

A: 可能的原因：
1. 还没有学习记录
2. 学习记录不完整
3. 单词学习记录为空

解决方法：
1. 确保在学习模块中正确记录数据
2. 进行一些学习活动
3. 刷新页面查看数据

### Q2: 质量得分计算不准确？

A: 检查：
1. 学习记录中的数据是否正确
2. 单词学习记录是否完整
3. 计算公式是否正确

### Q3: 质量建议不合适？

A: 检查：
1. 建议生成逻辑是否正确
2. 得分阈值是否合理
3. 建议内容是否可执行

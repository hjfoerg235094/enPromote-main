# 剧情化英语学习系统规划文档

## 一、项目概述

### 1.1 目标
将传统的单词练习融入连贯的剧情场景中，通过角色扮演和任务驱动的方式，提升学习者的学习兴趣和记忆效果。

### 1.2 核心特性
- **剧情连贯性**：每个场景包含多个关卡，关卡之间有剧情联系
- **角色扮演**：学习者在不同场景中扮演不同角色
- **多样化练习**：每个关卡包含不同类型的练习（拼写、听力、对话等）
- **进度追踪**：记录学习者在剧情中的进度和成就

## 二、系统架构

### 2.1 数据结构

#### 2.1.1 剧情场景（Story Scene）
```json
{
  "storyId": "SPY_STORY",
  "storyTitle": "特工行动：代号暗影",
  "storyDescription": "剧情简介",
  "totalChapters": 5,
  "chapters": [
    {
      "chapterId": 1,
      "title": "第一章：潜入",
      "scene": "机场入境",
      "aiRole": "海关官员",
      "userRole": "特工",
      "storyBackground": "剧情背景描述",
      "tasks": [...],
      "completionCriteria": {...},
      "nextChapterHint": "下一章提示"
    }
  ]
}
```

#### 2.1.2 任务类型（Task Types）
1. **对话任务（dialogue）**
   - 与AI角色进行对话练习
   - 需要使用指定单词
   - 记录对话轮次和单词使用情况

2. **拼写任务（spelling）**
   - 根据发音或上下文拼写单词
   - 系统提供即时反馈
   - 记录拼写正确率

3. **听力任务（listening）**
   - 听取AI角色的语音或文本
   - 回答相关问题或完成填空
   - 记录理解准确度

4. **阅读任务（reading）**
   - 阅读剧情相关的文本材料
   - 完成理解题或词汇练习
   - 记录阅读速度和理解度

## 三、实现步骤

### 第一阶段：数据结构设计（1-2天）

#### 步骤1.1：创建剧情场景配置文件
- 创建 `server/ai/storyScenes.json`
- 设计第一个完整剧情（如间谍主题）
- 包含5个章节，每个章节3-4个任务
- 定义不同类型的任务

#### 步骤1.2：设计数据库模型
- 创建用户剧情进度表（UserStoryProgress）
  - 字段：userId, storyId, currentChapter, completedChapters, totalScore
- 创建任务完成记录表（TaskCompletion）
  - 字段：userId, storyId, chapterId, taskId, completionTime, score

### 第二阶段：后端开发（3-4天）

#### 步骤2.1：创建剧情进度管理模块
- 文件：`server/modules/StoryProgress.js`
- 功能：
  - 获取用户剧情进度
  - 更新剧情进度
  - 计算章节完成度
  - 解锁新章节

#### 步骤2.2：创建任务管理器
- 文件：`server/modules/TaskManager.js`
- 功能：
  - 加载任务配置
  - 验证任务完成
  - 计算任务得分
  - 生成任务反馈

#### 步骤2.3：创建剧情路由
- 文件：`server/router/story.js`
- 接口：
  - GET `/api/story/list` - 获取可用剧情列表
  - GET `/api/story/:storyId` - 获取剧情详情
  - GET `/api/story/:storyId/progress` - 获取用户进度
  - POST `/api/story/:storyId/chapter/:chapterId/start` - 开始章节
  - POST `/api/story/:storyId/chapter/:chapterId/task/:taskId/complete` - 完成任务
  - GET `/api/story/:storyId/chapter/:chapterId/next` - 获取下一章

### 第三阶段：前端开发（4-5天）

#### 步骤3.1：创建剧情API接口
- 文件：`frontend/src/api/story.ts`
- 封装所有剧情相关的API调用

#### 步骤3.2：创建剧情状态管理
- 文件：`frontend/src/stores/storyStore.ts`
- 使用Pinia管理：
  - 当前剧情信息
  - 章节进度
  - 任务状态
  - 用户得分

#### 步骤3.3：创建剧情模式主界面
- 文件：`frontend/src/views/StoryMode.vue`
- 功能：
  - 显示剧情列表
  - 展示剧情简介和章节
  - 显示用户进度
  - 进入章节选择

#### 步骤3.4：创建章节界面
- 文件：`frontend/src/views/ChapterView.vue`
- 功能：
  - 显示章节背景和任务列表
  - 展示当前任务
  - 处理不同类型任务的交互
  - 显示任务完成反馈
  - 解锁下一章

#### 步骤3.5：创建任务组件
- 文件：`frontend/src/components/tasks/`
  - `DialogueTask.vue` - 对话任务组件
  - `SpellingTask.vue` - 拼写任务组件
  - `ListeningTask.vue` - 听力任务组件
  - `ReadingTask.vue` - 阅读任务组件

### 第四阶段：AI集成（2-3天）

#### 步骤4.1：设计剧情AI提示词
- 为不同场景设计AI角色提示词
- 为不同任务类型设计任务提示词
- 确保AI回复符合剧情设定

#### 步骤4.2：集成AI到任务系统
- 对话任务：使用AI进行角色扮演对话
- 拼写任务：使用AI生成拼写练习
- 听力任务：使用AI生成听力材料
- 阅读任务：使用AI生成阅读材料

### 第五阶段：测试与优化（2-3天）

#### 步骤5.1：功能测试
- 测试剧情流程
- 测试任务完成逻辑
- 测试进度保存

#### 步骤5.2：用户体验优化
- 优化界面交互
- 添加动画效果
- 优化反馈机制

#### 步骤5.3：性能优化
- 优化数据加载
- 优化AI响应速度
- 优化进度保存

## 四、示例剧情设计

### 剧情一：特工行动（5章）

#### 第一章：潜入
- 场景：机场入境
- 角色：特工 vs 海关官员
- 任务：
  1. 对话：应对身份盘问
  2. 拼写：填写入境卡
  3. 听力：理解入境规定

#### 第二章：接头
- 场景：酒店大堂
- 角色：特工 vs 酒店前台
- 任务：
  1. 对话：办理入住
  2. 阅读：阅读酒店安全须知
  3. 对话：与接头人暗号对接

#### 第三章：情报
- 场景：咖啡馆
- 角色：特工 vs 情报员
- 任务：
  1. 对话：交换情报
  2. 听力：听取情报内容
  3. 拼写：记录关键信息

#### 第四章：追踪
- 场景：街道
- 角色：特工 vs 路人/店主
- 任务：
  1. 对话：询问路线
  2. 阅读：阅读地图和路标
  3. 听力：听取追踪目标信息

#### 第五章：撤离
- 场景：火车站
- 角色：特工 vs 检票员
- 任务：
  1. 对话：购买车票
  2. 拼写：填写车票信息
  3. 听力：听取广播通知

### 剧情二：餐厅大冒险（5章）

#### 第一章：求职
- 场景：餐厅面试
- 角色：求职者 vs 经理
- 任务：
  1. 对话：自我介绍
  2. 阅读：阅读职位描述
  3. 对话：回答面试问题

#### 第二章：培训
- 场景：餐厅后厨
- 角色：新员工 vs 厨师长
- 任务：
  1. 听力：听取操作规范
  2. 拼写：记录菜名和食材
  3. 对话：询问工作细节

#### 第三章：首日
- 场景：餐厅大厅
- 角色：服务员 vs 顾客
- 任务：
  1. 对话：接待顾客
  2. 听力：听取顾客需求
  3. 拼写：记录点单

#### 第四章：危机
- 场景：餐厅
- 角色：服务员 vs 投诉顾客
- 任务：
  1. 对话：处理投诉
  2. 阅读：阅读菜单说明
  3. 对话：提供解决方案

#### 第五章：晋升
- 场景：餐厅办公室
- 角色：资深员工 vs 经理
- 任务：
  1. 对话：汇报工作
  2. 阅读：阅读评估报告
  3. 对话：讨论未来发展

# 口语评测模块设计文档

## 1. 概述

口语评测模块旨在帮助用户提升英语口语能力，通过技术手段提供智能化的发音评测和情景化对话练习。模块主要包含两大核心功能：

1. **发音评测**：基于科大讯飞免费版口语评测 API，对用户发音进行准确率、流利度等多维度评分
2. **口语对话**：基于 ChatGLM-6B 开源模型，提供英文情景化对话，为用户提供真实的口语交流场景

## 2. 功能模块设计

### 2.1 发音评测功能

#### 2.1.1 功能描述
用户通过前端录制英语发音，系统将语音文件上传至后端，后端调用科大讯飞口语评测 API 进行多维度评分，包括：
- 发音准确度
- 流利度
- 完整度
- 语速
- 声调/语调

系统根据评分结果提供个性化改进建议，帮助用户提升发音水平。

#### 2.1.2 技术架构

**前端部分**：
- 使用 Web Audio API 进行音频录制
- 支持实时波形显示
- 提供录音控制（开始、暂停、停止、重录）
- 音频格式：PCM/WAV，采样率 16kHz，单声道
- 上传前进行音频压缩和格式转换

**后端部分**：
- 接收前端上传的音频文件
- 调用科大讯飞口语评测 API
- 处理 API 返回的评分数据
- 生成个性化改进建议
- 保存评测记录到数据库

#### 2.1.3 API 设计

**科大讯飞口语评测 API**：
- 使用免费版 API
- 支持单词、句子、段落评测
- 返回详细评分和建议

**后端接口**：

```
POST /api/speaking/evaluate
Content-Type: multipart/form-data

参数：
- audioFile: 音频文件（WAV 格式）
- text: 待评测的文本内容
- level: 评测级别（word/sentence/paragraph）
- category: 评测类别（read_sentence/read_word）

返回：
{
  "code": 200,
  "message": "success",
  "data": {
    "overallScore": 85,  // 总分（0-100）
    "accuracy": 90,      // 准确度（0-100）
    "fluency": 80,       // 流利度（0-100）
    "integrity": 95,      // 完整度（0-100）
    "details": [          // 详细评分
      {
        "word": "hello",
        "score": 90,
        "phonemeScores": [...]
      }
    ],
    "suggestions": [      // 改进建议
      "注意元音 /ə/ 的发音",
      "提高语速的稳定性"
    ]
  }
}
```

#### 2.1.4 评测流程

1. 用户选择要练习的文本（单词、句子或段落）
2. 前端显示文本和发音提示
3. 用户点击"开始录音"按钮
4. 前端录制用户语音
5. 用户点击"停止录音"按钮
6. 前端将音频文件上传到后端
7. 后端调用科大讯飞 API 进行评测
8. 后端处理评分数据并生成建议
9. 前端展示评分结果和改进建议
10. 用户可以选择重新录音或继续练习下一个

#### 2.1.5 数据库设计

**评测记录表（SpeakingEvaluation）**：
```javascript
{
  userId: String,              // 用户ID
  text: String,               // 评测文本
  level: String,              // 评测级别
  category: String,           // 评测类别
  audioUrl: String,           // 音频文件URL
  overallScore: Number,        // 总分
  accuracy: Number,           // 准确度
  fluency: Number,           // 流利度
  integrity: Number,          // 完整度
  details: Array,            // 详细评分
  suggestions: Array,         // 改进建议
  createTime: Date,          // 创建时间
  updateTime: Date           // 更新时间
}
```

### 2.2 口语对话功能

#### 2.2.1 功能描述
基于 ChatGLM-6B 开源模型，提供英文情景化对话功能。系统预设多种真实生活场景（如：餐厅点餐、机场问询、求职面试等），用户可以在这些场景中进行口语对话练习，提升实际交流能力。

#### 2.2.2 技术架构

**前端部分**：
- 场景选择界面
- 对话界面（聊天式交互）
- 语音输入/文本输入切换
- 实时语音转文字（可选）
- 对话历史记录展示
- 对话评价和反馈

**后端部分**：
- ChatGLM-6B 模型部署和管理
- 场景化对话逻辑处理
- 上下文维护
- 对话记录保存
- 对话质量评估

#### 2.2.3 场景设计

预设场景列表：

1. **日常交流**
   - 自我介绍
   - 问候与告别
   - 表达感谢与道歉

2. **餐厅场景**
   - 点餐
   - 投诉与建议
   - 结账

3. **出行场景**
   - 机场问询
   - 酒店入住
   - 问路与

4. **购物场景**
   - 商品咨询
   - 讨价还价
   - 退换货

5. **工作场景**
   - 求职面试
   - 会议讨论
   - 汇报工作

6. **学习场景**
   - 课堂提问
   - 小组讨论
   - 学术交流

每个场景包含：
- 场景描述
- 常用句型
- 对话引导
- 难度级别（初级/中级/高级）

#### 2.2.4 API 设计

**后端接口**：

```
POST /api/speaking/dialogue/start
Content-Type: application/json

参数：
- scenario: 场景ID
- level: 难度级别

返回：
{
  "code": 200,
  "message": "success",
  "data": {
    "dialogueId": "xxx",
    "scenario": "restaurant",
    "level": "intermediate",
    "opening": "Welcome to our restaurant. How can I help you today?",
    "context": {...}
  }
}
```

```
POST /api/speaking/dialogue/message
Content-Type: multipart/form-data

参数：
- dialogueId: 对话ID
- audioFile: 音频文件（可选）
- text: 文本消息（可选）
- mode: 输入模式（audio/text）

返回：
{
  "code": 200,
  "message": "success",
  "data": {
    "message": "I'd like to order a steak, please.",
    "response": "Certainly! How would you like your steak cooked?",
    "feedback": {
      "fluency": 85,
      "grammar": 90,
      "vocabulary": 80,
      "suggestions": ["尝试使用更自然的表达方式"]
    },
    "context": {...}
  }
}
```

```
POST /api/speaking/dialogue/end
Content-Type: application/json

参数：
- dialogueId: 对话ID

返回：
{
  "code": 200,
  "message": "success",
  "data": {
    "summary": "本次对话练习了餐厅点餐场景...",
    "score": 88,
    "improvements": [...]
  }
}
```

#### 2.2.5 对话流程

1. 用户选择场景和难度级别
2. 系统初始化对话，显示开场白
3. 用户通过语音或文本输入回复
4. 系统调用 ChatGLM-6B 生成回复
5. 系统对用户输入进行评估（如果是语音）
6. 显示系统回复和评估结果
7. 重复步骤 3-6，直到对话结束
8. 系统生成对话总结和改进建议

#### 2.2.6 数据库设计

**对话记录表（SpeakingDialogue）**：
```javascript
{
  userId: String,              // 用户ID
  scenario: String,           // 场景ID
  level: String,              // 难度级别
  messages: [                // 对话消息
    {
      role: String,           // user/assistant
      content: String,        // 消息内容
      audioUrl: String,      // 音频URL（如果是语音）
      timestamp: Date,       // 时间戳
      feedback: {            // 评估反馈（仅用户消息）
        fluency: Number,
        grammar: Number,
        vocabulary: Number,
        suggestions: Array
      }
    }
  ],
  score: Number,              // 对话总分
  summary: String,           // 对话总结
  improvements: Array,         // 改进建议
  createTime: Date,          // 创建时间
  updateTime: Date           // 更新时间
}
```

3. 输入控制区
   - 语音/文本输入切换
   - 录音按钮和波形显示
   - 文本输入框
   - 发送按钮

4. 评估反馈区
   - 实时显示评估结果
   - 流利度、语法、词汇评分
   - 改进建议提示

5. 对话总结区
   - 对话结束后显示
   - 总体评分
   - 改进建议列表
   - 可保存对话记录

## 4. 技术实现要点

### 4.1 音频处理

**前端音频处理**：
- 使用 MediaRecorder API 录制音频
- 音频格式转换：PCM → WAV
- 采样率：16kHz
- 位深度：16-bit
- 声道：单声道
- 音频压缩：使用 Web Audio API 进行压缩

**后端音频处理**：
- 接收并验证音频文件
- 格式转换（如需要）
- 音频质量检查
- 调用评测 API
- 音频文件存储（云存储或本地）

### 4.2 科大讯飞 API 集成

**API 配置**：
- 使用免费版 API
- 配置 AppID 和 API Key
- 设置回调地址
- 配置评测参数

**评测参数**：
- 评测模式：单词/句子/段落
- 评测类别：朗读/跟读
- 评分维度：准确度、流利度、完整度

**错误处理**：
- API 调用失败处理
